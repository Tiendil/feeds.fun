# Feeds Fun specification overview

## Goal of the document

This document lists Feeds Fun specification documents and specification directories, and briefly describes their purpose.

## Scope

The scope of this specification is limited to the specification index.

Detailed requirements for individual specifications are out of scope except for brief descriptions needed to keep the index useful.

## Specification directories

- `spec/` contains all project specifications used by depmesh governance rules.
- `spec/backend_architecture/` contains specifications related to backend architecture, entities, tests, and errors.
- `spec/frontend_architecture/` contains specifications related to frontend architecture and tests.
- `spec/documentation/` contains specifications related to repository documentation artifacts.
- `spec/meta/` contains specifications related to requirements for specification documents.

## Specification documents

- `spec/intro.md` is this file and indexes all specification documents.
- `spec/dictionary.md` defines Feeds Fun and dependency metadata terms shared by multiple specifications.
- `spec/meta/general.md` defines general rules for project specification documents.
- `spec/backend_architecture/modules_layout.md` describes backend package layout and ownership boundaries.
- `spec/backend_architecture/entities.md` describes backend entity and data structure architecture.
- `spec/backend_architecture/errors.md` describes backend error and warning architecture.
- `spec/backend_architecture/tests.md` describes backend pytest test placement.
- `spec/frontend_architecture/modules_layout.md` describes frontend source layout and ownership boundaries.
- `spec/frontend_architecture/tests.md` describes frontend Vitest test placement.
- `spec/documentation/readme.md` describes repository README expectations.
- `spec/documentation/changelog.md` describes changelog artifact expectations.
